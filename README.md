# Портфолио страница

Персональный сайт‑портфолио разработчика (веб‑разработка + DevOps). Ресурс демонстрирует профиль автора, опыт и портфель проектов с быстрым доступом к демонстрациям и репозиториям.

## Назначение

- Показать специализацию автора и профессиональные компетенции.
- Представить проекты в виде карточек (каталог) и детальных страниц («Подробнее»).
- Дать удобный канал связи через форму обратной связи.

## Структура проекта

Корневые страницы:

- `index.html` — главная страница (hero, избранные проекты, сертификаты, навыки).
- `about.html` — страница «Обо мне» (ФИО, образование, опыт).
- `projects.html` — страница «Проекты» (каталог карточек всех проектов).
- `contacts.html` — страница «Контакты» (форма обратной связи и ссылки на соцсети).

Детальные страницы проектов (по ссылке «Подробнее»):

- `project-portfolio.html`
- `project-focuspod.html`
- `project-estateai.html`
- `project-realestate.html`
- `project-is-meropriyatiya.html`
- `project-tumar-ai.html`
- `project-ulan-ude.html`
- `project-bilet-5.html` — проект **We are sysadmins**

Разметка, стили и логика:

- `style.css` — стили и адаптивная вёрстка.
- `script.js` — генерация карточек проектов/сертификатов/навыков и логика интерфейса (mobile menu, аккордеон опыта, modal PDF, отправка формы).
- `config.js` — конфигурация для Telegram (секреты).
- `config.example.js` — пример конфигурационного файла.

Статические материалы:

- `images/` — изображения проектов и оформления.
- `documents/` — PDF‑материалы (для модального окна просмотра).
- `sertificat/` — сертификаты (картинки/файлы).

## Технологии

- HTML5 / CSS3.
- JavaScript (vanilla), динамическая генерация контента из массивов данных.
- Адаптивная вёрстка (Grid/Flex, медиазапросы).
- Three.js — декоративный 3D‑фон на главной странице.
- Font Awesome — иконки.
- Telegram Bot API — отправка сообщений с формы контактов.

## Отправка формы (Telegram)

На странице `contacts.html` используется форма, которая отправляет данные через Telegram Bot API из `script.js`.

В проекте конфигурация вынесена в `config.js`.

Важно:
- `config.js` содержит секреты (токен бота и chatId) и в публичные репозитории обычно не коммитится.
- Перед публикацией/демонстрацией замените `config.example.js` на `config.js` и укажите свои значения.

Пример структуры `config.example.js`:

```js
const TELEGRAM_CONFIG = {
  botToken: 'YOUR_BOT_TOKEN_HERE',
  chatId: 'YOUR_CHAT_ID_HERE'
};
```

## Портфель проектов

Карточки на `projects.html` генерируются в `script.js` на основе `projectsData`.
Для части проектов карточка «Подробнее» ведёт на именные страницы `project-*.html` (например, `project-bilet-5.html` для **We are sysadmins**), а также используются внешние ссылки для `Live Demo` и `Code`.

## Примечания по запуску

Проект — статический сайт. Для корректной работы отправки формы и запросов из браузера рекомендуется запускать его через локальный сервер/хостинг (а не только открывать `index.html` из `file://`).
